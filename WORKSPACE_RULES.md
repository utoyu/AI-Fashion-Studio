# Global Workspace Rules for AI Fashion Studio

## 1. Tech Stack & Framework
- **Core:** Next.js (App Router strictly, NO Pages router).
- **Language:** TypeScript strictly. Use strong typing for all API requests and responses.
- **Styling:** Tailwind CSS. 
- **UI Components:** Shadcn UI and Radix UI primitives (as generated by v0). Do not introduce other UI libraries like Material-UI or Ant Design.
- **Icons:** Lucide React.

## 2. Agent Behavior & Coding Standards
- **Protect the UI (CRITICAL):** The current UI was generated by v0 and is highly polished. DO NOT rewrite, delete, or drastically alter the layout or Tailwind classes of existing UI components unless explicitly instructed to fix a bug. Focus ONLY on wiring up state and logic.
- **Incremental Changes:** When implementing features, only modify the specific functions or state variables needed. Avoid touching unrelated code.
- **API First:** For backend logic, always use Next.js Route Handlers (`app/api/...`). Do not attempt to spin up a separate Express or Python server. All logic must be serverless-ready for Vercel deployment.
- **Logging:** Add clear `console.log` statements in API routes for external API calls (Fal.ai and Gemini) to help with debugging in the Vercel console.

## 3. External Integrations (Phase 1)
- **Fal.ai Integration:** Use `@fal-ai/client`. Ensure proper payload construction for `idm-vton`.
- **Gemini Integration:** Use Google's official `@google/genai` SDK. Instruct the model to return concise marketing copy based on the garment category and generated image.
- **Error Handling:** All API routes must have `try/catch` blocks and return appropriate HTTP status codes (400, 500) with clear JSON error messages. Never crash the frontend silently. Handle loading and error states gracefully in the UI.

## 4. Work Flow
- Before writing code, briefly output the file paths you intend to modify.
- If an NPM package needs to be installed, execute the terminal command first, wait for success, and then write the code.
