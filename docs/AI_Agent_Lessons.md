# 📔 经验总结与规避指南：Windows 环境下的文件操作

为了避免再次发生“路径未找到”或“文件拷贝失败”的错误，我总结了以下技术要点作为后续操作的强制标准：

## 1. 目录创建的原子性 (Atomic Directory Creation)
*   **问题**：之前使用的 `mkdir -p` 在某些环境下可能由于文件系统句柄延迟，导致紧随其后的 `cp` 命令找不到刚创建的目录。
*   **纠正方案**：改用 PowerShell 原生命令 `New-Item -ItemType Directory -Force`，并显式等待命令返回成功后再执行文件移动。

## 2. 路径完整性校验 (Path Integrity)
*   **问题**：在执行多行 `cp` 命令时，长路径可能由于行内包含空格或字符编码问题被截断。
*   **纠正方案**：
    *   **必须** 对所有路径使用双引号包裹。
    *   **必须** 优先使用绝对路径而非相对路径。
    *   **操作前校验**：在大规模移动前，先执行 `Test-Path` 验证目标根路径。

## 3. 错误预测与预防逻辑 (Error Prevention)
*   **行为调整**：
    *   **不再盲目批量执行**：对于涉及多个子目录的操作，采用“先建总目录 -> 验证 -> 再建子目录 -> 验证 -> 搬运文件”的阶梯式逻辑。
    *   **引入重试机制**：如果第一次 `cp` 失败，会自动检查是由于目录缺失还是源文件不存在，并即时修复路径。

## 4. 本次案例复盘 (Case Post-mortem)
*   **触发点**：在创建 `batch2` 家族目录后，立即搬运 10+ 张图片。
*   **失败原因**：系统还没来得及为新创建的子目录分配索引，导致 `cp` 抛出 `DirectoryNotFoundException`。
*   **下次规避**：将创建目录与文件搬运分拆到不同的命令调用中，确保操作系统层面已完成 I/O 同步。

## 5. 复杂仪表盘布局标准 (Dashboard Layout Standards)
*   **挑战**：在 90vh 的大屏详情对话框中，如何保持多行、多区域的精确水平对齐（如角度图与售后信息对齐）。
*   **经验**：
    *   **容器解耦**：采用“固定 Header + 固定 Footer + `flex-1` 滚动 Body”的模式，防止布局因内容过多而溢出视口。
    *   **栅格布局**：Body 内部统一使用 `grid-cols-12`，而非嵌套过深的 Flex 容器。这能确保不同板块（如 6 栏 vs 6 栏）在物理像素上绝对对齐。

## 6. 交互冲突处理 (Interaction Conflict Resolution)
*   **挑战**：素材格子同时需要“单击预览”和“双击编辑”，默认事件会产生冲突。
*   **经验**：
    *   **防抖机制**：在 `onClick` 中设置 `setTimeout`（约 250ms），如果在计时结束前触发了 `onDoubleClick`，则通过 `clearTimeout` 取消单击任务。
    *   **类型分发**：在多态列表（Polymorphic List）中，事件处理器必须第一时间校验 `item.type`。不要在通用处理器中混入类型特定的 UI 逻辑，应在入口处就分发给不同的模态框。

## 7. 列表编辑的状态同步 (State Sync in List Editors)
*   **挑战**：在主列表页弹出的详情页中修改数据，如何安全地存回原始数组。
*   **经验**：
    *   **原子化更新**：使用 `prev.map(a => a.id === editingAsset.id ? { ...editingAsset, modifier: "Admin" } : a)` 确保更新是不可变的（Immutable）且原子化的。
    *   **元数据补全**：在保存逻辑中，始终考虑自动补全 `modifyTime` 和 `modifier`，而非依赖外部输入。

## 8. 跨页面数据流与异步边界 (Cross-page Data Flow)
*   **挑战**：在从“摄影室”跳转到“智能精修”时，如何确保图片对象跨页面实时传递且不解析错误。
*   **经验**：
    *   **带参导航逻辑**：优先使用 `useSearchParams()` 配合 `encodeURIComponent` 实现素材地址的传递。
    *   **Suspense 强制隔离**：在 Next.js (App Router) 中，任何触发搜索参数解析的逻辑 **必须** 包裹在 `<Suspense>` 中。否则，在生产环境构建阶段会因为缺少客户端运行时，导致页面级错误。
    *   **多源状态机同步**：设计组件状态时，需统一“本地上传 (File/Blob CLI)”与“远程参数 (URL String)”的入口。建立 `workingImage` 统一指针，通过 `useEffect` 实时修正来源优先级。
