# 🕒 项目路线图与更新日志 (CHANGELOG.md)

## [Unreleased] - 2026-03-10
### 🚀 Features (新增功能)
* **事件驱动架构 (Event-Driven Architecture) 上线:** 正式废弃前端直接请求生图 API 的同步阻塞模式。
* **Supabase Realtime 接入:** 前端 `ManualRetouchPage` 成功集成 WebSocket 实时订阅，实现不刷新页面自动感知 AI 任务进度 (`pending` -> `processing` -> `completed`)。
* **自动化任务分发 (Task Queue):** 在 Supabase 创建了核心流转表 `ai_tasks`。
* **Python Daemon 守护进程测试通过:** 编写并跑通了本地 `worker.py`，实现全自动的“轮询发现新任务 -> 抢单修改状态 -> 模拟处理 -> 结果回传”链路。

### 🛠️ Refactoring (重构)
* 解除了 Supabase `ai_tasks` 表的 RLS (行级安全) 限制（原型测试阶段），打通前端监听的权限阻断问题。

## [2026-03-08] 全站云端同步与 SSR 韧性增强
### ✨ 新特性 (New Features)
- **Supabase 云端直传 (OSS Integration)**：全站图片上传逻辑从本地 `Blob` 彻底迁移至 Supabase Storage。实现了“素材库、电商组图、一键精修、模特定制、摄影室”五大核心板块的云端持久化存储，解决了 AI 生图引擎对本地临时 URL 的排斥问题。
- **存储鲁棒性优化**：在 `lib/storage.ts` 中引入了基于 MIME 类型的自动后缀识别与 ASCII 文件名脱敏机制，彻底解决了中文文件名导致的 400 存储错误。

### 🛠️ 优化与修复 (Fixes & Optimizations)
- **全局 Hydration 注入修复**：解决了 Next.js SSR 模式下因直接在 `useState` 中读取 `localStorage` 导致的“水合失败 (Hydration Mismatch)”报错。通过引入 `isHydrated` 状态守卫，确保数据载入仅在客户端挂载后执行，保障了首屏渲染的平滑。
- **配置持久化增强**：对“电商组图”及“模特定制”页面的所有业务参数（肤色、体型、平台选择、产品属性等）进行了客户端持久化重构，确保页面刷新后工作流状态不丢失。

### ✨ 新特性 (New Features)
- **快捷工具架构分立**：将原先混合的“图片工具”拆分为独立且语义明确的 **“一键精修” (One-Click Retouch)** 和 **“手搓精修” (Manual Retouch)**。前者面向单图极速处理，后者支持原图与参考图/版式的双轨并行深度重构。
- **色彩与品牌一致性 (Theme Consistency)**：对全站所有控制台页面（包含 KOC、摄影室、模特定制等）进行了色彩审计。彻底移除了各页面历史遗留的、硬编码的紫/蓝/橙色调，统一向核心品牌主题色 `primary` (青色) 收敛。
- **存档中心 (Archive Center)**：设立了专属页面存放已下线或实验性质的组件（包含原“智能精修”页面及旧版补全实验室），保证主干流程的极简与清晰。

### 🛠️ 优化与修复 (Fixes & Optimizations)
- **全局 Header 视觉污染修复**：修复了因 `scale` 和透明度配置不当，导致资产库等页面顶部操作区出现“蓝色底纹泄露 (Blue Tint)”的渲染 Bug，统一了 `bg-slate-50/50` 及 `bg-white/80` 的遮罩标准。
- **导航层级同步与防御性布局**：统一定制了各功能页面的 Header 右侧安全距离 (`pr-[400px]`)，防止局部操作按钮与全局悬浮控件（配额、主题切换）发生交互穿透和层叠冲突。

## [2026-03-06] 电商组图核心重构与工作流打通
### ✨ 新特性 (New Features)
- **精英型录全面升级**：将原有的泛社交媒体文案生成器“精英型录”彻底重命名并重构为垂直领域的**“电商组图 (E-Commerce Catalog)”**。
- **动态电商矩阵**：构建了专业的主推图及副参考图组合，支持一键生成符合不同电商及内容平台要求的 16:9、3:4 等高分辨率营销长图。
- **本地+云端资产穿梭**：支持直接上传本地实拍图，新增 **[ 关联属性提取 ]** 创新功能——允许用户为本地图片独立挂接系统内标准资产的物理摘要（面料、洗护、工艺点等），而不会强行覆盖主图的视觉呈现。
- **全局词汇与导航打通**：统一主页、各类菜单中模块名称至“电商组图”，并将“素材库”导航层级提升至核心应用首位梯队（摄影室之后），强化素材底座地位。

## [2026-03-05] 智能素材库与 Agent 进化
### ✨ 新特性 (New Features)
- **多态详情系统**：为衣物、模特、背景分别设计了独立的 90vh 全屏详情编辑页，支持根据类型动态渲染属性（成分、角度图、生物学数据、环境光影等）。
- **角度图管理**：实现了衣物素材的主图、背图、侧图、顶/底图的批量上传、实时预览与管理功能。
- **商务资产包**：批量导入了 14 张高品质商务男装品牌相关资产，包含模特、外景办公场景及高支衬衫。

### 🛠️ 优化与修复 (Fixes & Optimizations)
- **摄影室工作流重组**：将混杂的功能块划分为“基础、主体、细节”三大语义化模块，降低了认知负荷。
- **Prompt 智能实验室**：引入了可视化标签云系统，支持“电影感、伦勃朗光”等专业提示词一键添加。
- **跨模块数据闭环**：打通了 `Photo Studio -> Smart Retouch` 的带参跳转链路，实现图片素材的秒级流转。
- **稳定性修复**：解决了 `Smart Retouch` 页面因 URL 异步传参导致的渲染崩溃与标签嵌套问题。
- **交互解耦**：解决了资产格子单/双击冲突问题，平衡了预览与编辑的操作逻辑。
